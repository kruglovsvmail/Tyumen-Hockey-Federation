# Handoff: Сайт Федерации Хоккея Тюмени

## Overview
Промо-сайт Тюменской городской общественной организации «Федерация Хоккея»: главная страница-дашборд, разделы «О федерации», «Чемпионаты» (3 дивизиона со своими под-дивизионами, командами и игроками), «Медиа» (фотоальбомы + видео) и «Сотрудничество». Визуальный стиль — «ледяное стекло» (glassmorphism) на фоне анимированной 3D-хоккейной арены, светлая тема, шрифты Aire Exterior (заголовки) + Manrope (текст).

## About the Design Files
Файлы в этом пакете — **HTML-прототип**, созданный как референс дизайна и поведения, а не production-код для копирования. Это один файл `Федерация Хоккея.dc.html`, собранный на внутреннем движке дизайн-инструмента (кастомные теги `<x-dc>`, `<sc-for>`, `<sc-if>`, `data-props` и т.д. — не переносите их в продакшен). Задача разработчика — **воссоздать этот дизайн** в целевом стеке.

Рекомендуемый стек с учётом целей (GitHub-репозиторий, деплой на Timeweb, дальнейшее развитие логики — сезоны, статистика, CMS для новостей/фото):
- Frontend: **React + Vite** (или Next.js, если нужен SSR/SEO для новостей) + TypeScript, стилизация — CSS-in-JS или Tailwind (дизайн сейчас на инлайн-стилях с CSS-переменными — переменные из раздела «Design Tokens» ниже переносятся почти 1:1 в тему/токены).
- Backend/CMS: сайт по факту управляет большим объёмом структурированных данных (дивизионы → команды → игроки, календарь, новости, альбомы, партнёры) — на это стоит завести простую админку/CMS (Strapi/Directus как self-hosted на Timeweb, либо своя API на Node/NestJS + Postgres) вместо хардкода данных в коде.
- Хостинг: статическая сборка (Vite build) на Timeweb Static Hosting, либо Node-сервер на Timeweb Cloud, если выбран Next.js/SSR.

## Fidelity
**High-fidelity.** Все цвета, шрифты, отступы, эффекты стекла и анимации в прототипе финальные и согласованы с заказчиком — реализовывать пиксель-в-пиксель.

## 3D Ice Rink Background
Ключевая фича: фон — стилизованная 3D-хоккейная арена, построенная на чистом CSS 3D-transform (без WebGL/three.js), которая **облетает камерой** при смене раздела.
- Контейнер с `perspective: 1250px`, внутри «риг» с `transform-style: preserve-3d`, базовый вид — `rotateX(63deg)` (вид на лёд под углом).
- При переходе между разделами меняется `rotateZ` + `scale` + `translateY` рига с transition ~1s cubic-bezier — создаёт эффект облёта арены камерой к разным «зонам». Целевые трансформы на каждый раздел (см. `Z` map в логике):
  - `home`: `translateY(15vh) rotateX(63deg) rotateZ(0deg) scale(1.05)`
  - `fed` (о федерации): `translateY(21vh) rotateX(59deg) rotateZ(36deg) scale(1.34)`
  - `champ` (чемпионаты): `translateY(19vh) rotateX(61deg) rotateZ(-34deg) scale(1.28)`
  - `media`: `translateY(25vh) rotateX(57deg) rotateZ(116deg) scale(1.18)`
  - `partners`: `translateY(13vh) rotateX(65deg) rotateZ(-102deg) scale(1.12)`
- Лёд: скруглённый прямоугольник с полупрозрачной заливкой (`--icefill1/2/3`) и бортами (`--boardsbrd`, свечение `--boardsglow`), разметка — центральная красная линия, две синие линии, вбрасывающие круги (4 угловых + центральный) с воротами на коротких бортах (два прямоугольных 3D-каркаса ворот с сеткой, построены на вложенных `rotateZ`/`rotateX`/`translate3d`).
- Полупрозрачный логотип федерации в центре льда.
- Лёгкий постоянный «дрейф» всей сцены (`@keyframes drift`, ±1.1deg rotateZ, 16s) — фон никогда не абсолютно статичен.
- Поверх — падающий «снег»: 22 маленьких кружка, двигаются по diagonal-траектории сверху вниз с разной длительностью/задержкой (`@keyframes fallA/fallB`), у каждой снежинки случайные `left`, `size`, `duration`, `delay`.
- Виньетка поверх сцены: `radial-gradient` от прозрачного к `var(--vig)` снизу, для читаемости контента.
- Обе фичи (снегопад, вся 3D-сцена) — опциональные тумблеры в панели тюнинга прототипа (`snowfall`, `showScene`), полезно оставить как feature-флаги и в продакшене (для слабых устройств/производительности).

## Interactions & Behavior
- **Навигация**: SPA-подобная — переключение «разделов» (`zone` state: home/fed/champ/media/partners) держит все экраны смонтированными (`position: fixed`, `visibility: hidden/visible` + `transform: translateY()`), с отдельными transition delay на скрытие/появление чтобы не было наложения двух экранов одновременно (уходящий уезжает за 0.22s, затем через 0.28s delay въезжает новый — тайминг подобран чтобы убрать "прыжки" при быстрых переключениях).
- **Header/Home dropdown menus**: `onMouseEnter`/`onMouseLeave` per menu-key, `menu` state хранит текущий открытый ключ; сам заголовок пункта меню (О федерации/Чемпионаты/Медиа) специально сделан некликабельным — переход возможен только через клик по конкретному пункту внутри выпадалки.
- **Матчи недели carousel**: `matchPage` state, `Math.ceil(count / perPage)` страниц, стрелки клэмпятся к границам (не зацикливаются).
- **Новости**: `openNews: {}` объект по индексу — toggle раскрытия конкретной новости.
- **Фотокарусель**: `album`/`slide` state, поддержка клавиатуры (глобальный `keydown` листener пока открыт альбом).
- **Раздел «О федерации» / «Медиа» / дивизион**: `fedTab`/`mediaTab`/`champTab` state с `tabIn` анимацией смены контента.
- **Дивизион**: `champClass` (master/lubitel/vip) → `champDiv` (индекс под-дивизиона, `null` = список) → `teamIdx` (индекс команды, `null` = страница дивизиона) — это стек навигации, «назад» сбрасывает соответствующий уровень.
- **Сезон**: `season` state (строка вида '2025/26'), сейчас переключает только лейбл — данные календаря/таблицы одинаковы для всех сезонов в прототипе; в проде должен подгружать реальные данные сезона.
- **Календарь дивизиона**: данные сгенерированы по 3 стадиям (Круг 1 / Круг 2 / плей-офф с 1/4-1/2-финалом) и просто склеены в один список для вкладки «Календарь» (без визуальной группировки — де-факто плоский список с колонкой «Стадия»).

## Технические ограничения (важно сохранить в проде)
- **Ни одна анимация/transition не использует `opacity`.** Все появления/скрытия/переключения сделаны через `transform` (`translateY`, `translateX`, `scale`) + `visibility`. Причина: элементы со стеклянным эффектом используют `backdrop-filter: blur()`, а в Chrome `backdrop-filter` перестаёт рендериться (пропадает размытие) на элементах, чья анимация одновременно трогает `opacity`. Если это ограничение в целевом браузерном движке будет снято/неактуально — можно ослабить, но по умолчанию переносите паттерн «только transform» для всех анимаций элементов с блюром.
- Все границы (`border`) на плашках/карточках/кнопках — ровно `1px solid`, единый цвет `var(--head)`-based (см. токены), никаких 2px+ бордеров у UI (2px+ есть только в декоративной SVG-like 3D-сцене арены — это часть иллюстрации, не UI).
